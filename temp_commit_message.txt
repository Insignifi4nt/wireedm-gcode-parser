fix(canvas-grid,markers): align text labels with correct transform (Strategy A)

Resolves coordinate system mismatch that caused grid labels and point markers
to render misaligned due to double-transform error in text positioning logic.

## Problem Fixed
- Text rendering applied both custom transform AND coordinate conversion
- Result: double transformation causing position errors scaling with zoom/offset
- Grid labels appeared misaligned with grid lines across zoom levels
- Point marker labels (START/END/P1/P2) positioned incorrectly

## Solution: Strategy A (Screen-Space Text Rendering)
- **CanvasGrid.js**: Remove `applyTextTransform()`, use identity transform + `viewport.worldToScreen()`
- **MarkerRenderer.js**: Remove `applyTextTransform()`, use identity transform + `viewport.worldToScreen()` 
- **Text positioning**: Single coordinate conversion, no double-transform
- **DPI handling**: Fix cssToWorld math from `px * dpr / zoom` to `px / (dpr * zoom)`

## Additional Fixes
- **CoordinateTransforms.js**: Fix grid line calculation boundary symmetry
- **PathHighlights.js**: Fix DPI math in all cssToWorld functions
- **Documentation**: Update coordinate system usage guidelines

## Testing & Validation
- Validated mathematical accuracy across zoom levels (0.5x to 10.0x)
- Round-trip coordinate conversion accuracy < 0.001 error
- Build passes without errors or warnings
- All coordinate calculations now mathematically consistent

## Architecture Impact
- Eliminates coordinate system confusion
- Simpler, more reliable text rendering approach
- Maintains backward compatibility with geometry rendering
- Prevents similar double-transform issues in future

Fixes text alignment across all zoom levels with clean, maintainable solution.
